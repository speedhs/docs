**Definitions**

-   **Churn Rate**: Number of members who left during the period divided
    by the total number of members at the start of the period

-   **Revenue Churn**: MRR lost from departing members

-   **Net Revenue Churn**: MRR lost from departing members minus MRR
    gained from expansions

-   **At-Risk Member**: Member exhibiting two or more early warning
    signals

-   **Intervention Success**: Member identified as at-risk who remained
    after targeted retention efforts

**Implementation Notes**

This comprehensive churn reporting framework should be:

1.  **Automated**: Connect to CRM, billing, and member management
    systems for real-time data population

2.  **Interactive**: Deploy as an interactive dashboard with drill-down
    capabilities

3.  **Distributed**: Automatically delivered to key stakeholders on the
    5th of each month

4.  **Action-Oriented**: Include task assignment and tracking
    functionality

5.  **Comparable**: Maintain historical data for trend analysis

6.  **Predictive**: Incorporate machine learning models to improve
    forecasting accuracy

7.  **Visual**: Include data visualization for all key metrics

8.  **Collaborative**: Allow for comments and discussion threads on
    specific insights

9.  **Mobile-Friendly**: Accessible via mobile devices for executives on
    the go

10. **Secure**: Role-based access controls for sensitive financial data

This framework not only tracks churn but provides comprehensive insights
into retention effectiveness, financial impact, and actionable
recommendations to optimize Skelsa\'s member lifetime value and
community stability.
